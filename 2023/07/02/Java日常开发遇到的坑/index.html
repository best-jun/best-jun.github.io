Java日常开发遇到的坑

2023/7/2

一.空指针问题

（1）包装类型空指针问题


Integer value = null;
int res = value + 1;

（2）级联调用的空指针问题


//service.getMyService() 可能为空，会导致空指针问题
service.getMyService().work();

（3）Equals方法左边的空指针问题


String name = null;
System.out.print(name.equals("best-jun"));

（4）ConcurrentHashMap 这样的容器不支持 Key，Value 为 null。


Map map = new ConcurrentHashMap<>();
String key = null;
String value = null;
map.put(key, value); //报错

（5）集合，数组直接获取元素


int [] arr = null;
List list = null;
System.out.println(arr[0]); //空指针异常
System.out.println(list.get(0)); //空指针异常

（6）对象直接获取属性


Student s = null;
System.out.println(s.getAge()); //空指针异常

二.Integer缓存的问题


Integer a = 127;
Integer b = 127;
System.out.println("a==b:"+ (a == b));

Integer c = 128;
Integer d = 128;
System.out.println("c==d:"+ (c == d));

运行结果：


a==b:true
c==d:false

原理：默认情况下呢，Integer的缓存区间就是[-128, 127]，所以会出现这种情况。
另外，可以通过设置 JVM 参数加上 -XX:AutoBoxCacheMax=1000，来调整Integer缓存区间参数的

三.事务未生效的坑

  • 底层数据库引擎不支持事务
  • 在非public修饰的方法使用
  • rollbackFor属性设置错误
  • 本类方法直接调用
    
public class TransactionTest{
public void A(){
  //插入一条数据
  //调用方法B (本地的类调用，事务失效了)
  B();
}

@Transactional
public void B(){
  //插入数据
}
}
  • 异常被try…catch吃了，导致事务失效。
    
@Transactional
public void method(){
try{
  //插入一条数据
  insertA();
  //更改一条数据
  updateB();
}catch(Exception e){
  logger.error("异常被捕获了，那你的事务就失效咯",e);
}
}
